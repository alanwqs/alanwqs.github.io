# 一、转义不完全引发的漏洞

# 1、跨站脚本攻击

在用户的浏览器内运行非法的HTML标签或js进行的一种攻击。

影响：

  • 利用虚假表单骗取用户个人信息
  • 利用脚本窃取用户的cookie值
  • 显示伪造的文章或图片

# 2、SQL注入攻击

针对web应用使用的数据库，通过运行非法的SQL而产生的攻击，会导致个人信息的泄漏。

影响：

  • 非法查看或篡改数据库数据
  • 规避认证
  • 执行和数据库服务器关联的程序

# 3、OS命令注入攻击

通过web应用，执行非法的操作系统命令达到攻击的目的，只要能调用shell函数的地方就可能存在被攻击的风险

# 4、HTTP首部注入攻击

通过在响应首部字段内插入换行，添加任意响应头或主体的一种攻击，属于被动攻击模式。

影响：

  • 设置任何cookie信息
  • 重定向至任意url
  • 显示任意的主体

# 5、远程文件漏洞

当部分脚本内容需要从其他文件读入时，利用置顶外部服务器的url充当依赖文件，让读取文件之后，就可运行任意脚本的一种攻击

# 二、设计缺陷引发的漏洞

# 1、不正确的错误信息处理

开发者在处理错误时，暴露出对攻击者有用的信息，比如web应用抛出的错误信息，数据库抛出的错误信息等。

# 2、开放重定向

是一种对指定任意url开放重定向跳转的功能，假如跳转到某个具有恶意的web网站，那么用户就会被诱导到那个网站

# 三、会话管理疏忽漏洞

# 1、会话劫持

攻击者利用某种手段拿到用户的会话ID，并非法使用此会话ID

会话ID泄漏：

  • 通过非正规生成推测会话ID
  • 通过窃听或XSS攻击窃取会话ID
  • 通过会话固定攻击强行获取会话ID

# 2、跨站点请求伪造

攻击者利用已经设置好的陷阱，对已经完成认证的用户进行登录状态的获取，非法使用登录状态进行账户的操作

影响：

  • 通过已认证的用户更新设定信息
  • 通过已认证的用户购买商品
  • 通过已认证的用户在留言板发表评论等

# 四、其他安全漏洞

# 1、密码破解

  • 穷举法
  • 字典攻击

# 2、点击劫持

利用透明的按钮或链接，覆盖在正常的web网站上面，在用户不知情的情况下，点击链接进行内容访问的一种攻击手段。

# 3、Dos攻击

一种让运行中的服务呈停止状态的攻击，有时也叫服务停止攻击或拒绝服务攻击。

# 4、后门程序

是指开发设置的隐蔽的入口，可不按正常的操作访问受限的功能
