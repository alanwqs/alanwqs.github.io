    • 一、HTTP的瓶颈
      • 二、SPDY的目标
        • 三、Websocket协议

        # 一、HTTP的瓶颈

        • 一条连接只可以发送一个请求
        • 请求只能从客户端开始，客户端不可以接收除响应以外的指令
        • 请求/响应首部未经压缩就发送。首部信息越多延迟就越大
        • 发送冗长的首部，每次互相发送的首部造成的延迟浪费就越多
        • 可任意选择数据压缩格式，非强制压缩发送

        # 二、SPDY的目标

        spdy没有改写http协议，只是在应用层和传输层之间加入了会话层，通信中使用ssl

        使用SPDY协议之后，HTTP将获得以下的功能

        • 多路复用流
        • 请求优先级
        • 压缩HTTP首部
        • 推送功能
        • 服务器提示功能

        # 三、Websocket协议

        利用ajax和comet技术可以提升web的访问速度，但是只要是基于http协议就难免会遇到瓶颈。websocket协议是一套新协议以及API。

        Websocket协议的特点：

        • 推送功能
        • 减少通信量

        为了实现Websocket通信，需要用到HTTP的Upgrade首部字段，告知服务器协议发生改变，以达到握手的功能。
