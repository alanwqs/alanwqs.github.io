      • 三、使用cookie进行状态管理

      # 一、HTTP请求方法

      # GET

      TIP

      用于获取网络资源

      # POST

      TIP

      用于传输实体主体，比如提交表单数据

      # PUT

      TIP

      用于传输文件，像FTP一样，请求报文里包含文件内容，传输到指定位置

      # HEAD

      TIP

      用于获取报文首部，与GET方法一样只是不返回报文主体部分，用于确认URI的有效性及资源更新日期

      # DELETE

      TIP

      用于删除文件，与PUT相反，删除指定的文件

      # OPTIONS

      TIP

      用于删除文件，与PUT相反，删除指定的文件

      # TRACE

      TIP

      用于追踪路径，可以查询发送出去的请求是怎样被加工修改，篡改的。比如经过代理中转，可以确认链接过程的一系列操作。

      # CONNECT

      TIP

      要求与代理服务器连接是，用隧道协议。实现用隧道进行TCP通信。

      # 二、HTTP持久链接

      只要一端没有明确提出断开链接，则保持TCP的链接状态。好处是减少了TCP链接重复建立和断开造成的额外开销，减轻服务器的压力。

      管线化技术

      可以同时并行发送多个请求，不需要一个个等待响应

      # 三、使用cookie进行状态管理

      cookie技术通过在请求和响应中写入cookie信息控制客户端状态。

      第一次链接没有cookie，服务器响应之后会返回cookie写进浏览器，下次浏览器请求带上cookie，服务器就知道了是哪个客户端进行的请求了。
